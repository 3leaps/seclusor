# Release Notes — v0.1.2

**Release Date**: March 19, 2026  
**Status**: Go Signing Parity & Public Polish

## Summary

v0.1.2 completes the Ed25519 signing story for Go consumers and performs a focused public-repo polish pass to improve trust and consumability.

## What's New

### Go Signing Bindings (D12)

- Added Go bindings for the Ed25519 signing APIs introduced in v0.1.1
- Exposes idiomatic Go wrappers: `GenerateSigningKeypair`, `SigningPublicKeyFromSecretKey`, `Sign`, `Verify`, and `WipeBytes`
- Maintains the same defensive key material handling and error hygiene as the Rust implementation
- Includes comprehensive parity tests proving Rust and Go produce identical results for keys, signatures, and error cases

### Public Repo Polish (D12B)

- Trimmed stale unmatched license allowances from `deny.toml` so `cargo deny check licenses` output is quieter
- Updated top-level docs to reflect that Go signing now ships in v0.1.2
- Aligned short-form release docs (`CHANGELOG.md`, `RELEASE_NOTES.md`, this file) with the shipped Go signing surface

## Notes

- TypeScript signing bindings remain out of scope for this release
- The FFI contract for signing remains stable per ADR-0008
- Go secret-key wiping remains best-effort; use `WipeBytes` promptly after encrypting or otherwise consuming seed material
