# Release Notes - v0.1.0

**Release Date**: March 2026  
**Status**: Initial MVP (Rust rewrite)

## Summary

This is the first public release of seclusor as a Rust library-first secrets management tool using age encryption.

### Key Features

- **Library-first architecture** with clean separation: `seclusor-core`, `seclusor-crypto`, `seclusor-codec`, `seclusor-keyring`, `seclusor-ffi`
- **Two storage codecs**: Bundle (opaque, secure) and inline (`sec:age:v1:`) for git-friendly use
- **Secure CLI** including `seclusor run` for injecting secrets without exposing them in shell history or process lists
- **First-class bindings** for Go and TypeScript (via FFI/NAPI)
- **Safe by default**: redaction, stdout purity, size limits, permission checks
- Built-in documentation via `seclusor docs`

### Important Notes

- **Git storage of secrets** is supported but **not always recommended**. See [App Note 01: Git Storage of Armored Secrets](../docs/appnotes/01-git-armored-storage.md) for risk guidance by sensitivity level (low/medium/high, including glassbreak credentials).
- Primary consumer is the Lanyte platform for session attestation and secure credential handling.

### Changes

- Complete Rust rewrite from prior internal Go implementation
- Full feature parity with previous unpublished version plus improved library API
- Comprehensive security documentation and app notes
- Embedded documentation system

See `CHANGELOG.md` for full history and `docs/appnotes/` for usage guidance.
