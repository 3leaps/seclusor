# Release Notes — v0.1.6

**Release Date**: April 2026
**Status**: Inline Runtime Decryption Fix and Workflow Scenarios Guide

## Summary

v0.1.6 fixes a bug where inline-encrypted values (`sec:age:v1:`) were not
decrypted at runtime by `secrets run`, `secrets get`, and `secrets export-env`
when identity files were provided. It also adds a comprehensive workflow
scenarios guide to the embedded documentation.

## Bug Fix

### Inline-Encrypted Runtime Decryption (SC-012)

Prior to v0.1.6, runtime commands (`get`, `export-env`, `run`) would return
the raw `sec:age:v1:` ciphertext string instead of decrypting it when an
inline-encrypted secrets file was used with `--identity-file`. This made the
inline codec unusable for runtime workflows without a separate
`inline decrypt` step.

**Before (broken)**:

```bash
seclusor secrets run \
  --file secrets-inline.json \
  --identity-file ./identity.txt \
  --project myapp \
  -- env | grep MYAPP_
# MYAPP_DB_PASSWORD=sec:age:v1:YWdlLWVuY3J5cHRpb24ub3Jn...
```

**After (fixed)**:

```bash
seclusor secrets run \
  --file secrets-inline.json \
  --identity-file ./identity.txt \
  --project myapp \
  -- env | grep MYAPP_
# MYAPP_DB_PASSWORD=placeholder-synthetic-password-xyz789
```

The fix is in `seclusor-codec`'s `resolve_runtime_source` function, which now
passes provided identities through to the inline decryption path. The
`secrets get --reveal` command also correctly rejects inline ciphertext when
no identity is provided, with a clear error message.

## Documentation

### Workflow Scenarios Guide (SC-014)

New `docs/guides/scenarios/` directory with 7 end-to-end workflow guides:

| Slug                           | Guide                                                 |
| ------------------------------ | ----------------------------------------------------- |
| `scenarios/index`              | Overview, decision tree, identity setup paths         |
| `scenarios/bundle-credentials` | Init → set → bundle encrypt → delete plaintext → run  |
| `scenarios/inline-credentials` | Inline encrypt with git workflow and runtime use      |
| `scenarios/blob-encryption`    | Opaque file encryption for scripts, certs, configs    |
| `scenarios/ci-automation`      | GitHub Actions and GitLab CI with passphrase channels |
| `scenarios/team-recipients`    | Multi-recipient team workflows and member rotation    |
| `scenarios/quick-reference`    | "I want to..." command lookup table                   |

All guides are available via `seclusor docs show <slug>` and cross-linked
with existing concept docs (codecs, identity-and-recipients, cli-reference,
security, key-management) and app notes.

### Codecs Guide Fix

Fixed stale flag names in `docs/guides/codecs.md` — `--to-codec` and
`--file` updated to the correct `--from`/`--to` and `--input`/`--output`
flags matching the shipped CLI.
