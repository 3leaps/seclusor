# Release Notes — v0.1.4

**Release Date**: April 2, 2026
**Status**: Passphrase-Protected Identities, CLI Documentation, and Release Infrastructure

## Summary

v0.1.4 adds passphrase-protected identity files (the SSH-key-with-passphrase
equivalent for age identities), completes CLI help text coverage, and aligns
release asset naming with the org-wide convention for homebrew and scoop
distribution.

## What's New

### Passphrase-Protected Identity Files (SC-008)

Identity files can now be encrypted at rest using age's scrypt passphrase
mode. The secret key is protected while the public key remains visible in
a header comment for encryption without decryption.

```bash
# Generate a passphrase-protected identity (prompts twice):
seclusor keys age identity generate \
  --output ~/.config/seclusor/identity.txt --passphrase

# Use it — seclusor auto-prompts when it detects a protected file:
seclusor secrets run \
  --file secrets.age \
  --identity-file ~/.config/seclusor/identity.txt \
  --project demo \
  -- ./my-command

# CI/automation (no interactive prompt):
seclusor secrets run \
  --file secrets.age \
  --identity-file ~/.config/seclusor/identity.txt \
  --passphrase-env SECLUSOR_PASSPHRASE \
  --project demo \
  -- ./my-command
```

Four passphrase input channels are supported:

| Channel              | Flag                     | Use case              |
| -------------------- | ------------------------ | --------------------- |
| Interactive prompt   | `--passphrase`           | Developer workstation |
| Environment variable | `--passphrase-env VAR`   | CI/automation         |
| File                 | `--passphrase-file PATH` | Local dev             |
| Stdin pipe           | `--passphrase-stdin`     | Automation chaining   |

Passphrase channels are mutually exclusive. Passphrases are never accepted
as CLI argument values (would appear in process list and shell history).

**Security properties**:

- Passphrases held as `SecretString` (zeroized on drop) at all CLI boundaries
- Protected identity files enforce 0600 permissions on Unix, same as plaintext
  identities. Passphrase files (`--passphrase-file`) additionally enforce
  current-user ownership
- Size capped at 8 KiB before decryption attempt
- Malformed armor fails closed — no plaintext fallback
- Wrong passphrase produces a content-free error ("wrong passphrase or
  corrupted identity file")
- Empty passphrases rejected at all channels

Existing plaintext identity files work exactly as before — fully backward
compatible.

### CLI Help Text (SC-007 Part A)

Every clap argument across all 17 CLI structs now has help text. Running
`seclusor <command> --help` shows descriptions for all flags including
`--identity-file`, `--recipient`, `--passphrase-*`, `--value`, `--ref`,
and all others.

### Identity and Recipients Guide (SC-007 Part B)

New documentation at `docs/guides/identity-and-recipients.md` (also
available via `seclusor docs show identity-and-recipients`) explaining:

- What identity files and recipients are
- How they relate (encrypt with recipients, decrypt with identities)
- Passphrase-protected vs. plaintext identity files
- Generation, storage, and permission requirements by platform
- Multi-recipient encryption and team workflows

## Release Infrastructure

### Asset Naming Convention (DDR-0003)

Release assets now follow the org-wide `<app>-<os>-<arch>` naming convention
(bare binaries, no version prefix, no archive wrappers):

- `seclusor-linux-amd64`, `seclusor-linux-arm64`
- `seclusor-darwin-arm64`
- `seclusor-windows-amd64.exe`, `seclusor-windows-arm64.exe`

### Package Manager Distribution

- **Homebrew**: `brew install 3leaps/tap/seclusor`
- **Scoop**: `scoop bucket add 3leaps https://github.com/3leaps/scoop-bucket && scoop install seclusor`

### Makefile Improvements

- `update-homebrew-formula` and `update-scoop-manifest` targets with
  next-steps guidance, wired into `release-upload`
- `release-upload` now uploads provenance only (checksums, signatures,
  keys, notes); `release-upload-all` includes platform binaries

## Notes

- No breaking changes to the FFI contract or JSON schema in this release.
  New public APIs were added to `seclusor-keyring` and new CLI flags were
  added; see the changelog for details
- Rekeying remains a library-only API (CLI subcommand planned)
- One passphrase-protected identity per invocation (v0.1.x limitation;
  per-file passphrase mapping deferred to v0.2+)

See `CHANGELOG.md` for the structured change list.
