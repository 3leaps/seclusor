# Release Notes — v0.1.5

**Release Date**: April 2, 2026
**Status**: Blob Encryption for Opaque Files

## Summary

v0.1.5 adds `secrets blob encrypt` and `secrets blob decrypt` for
encrypting arbitrary files using age identity/recipient keys. Unlike
bundle and inline commands, blob treats input as opaque bytes — no JSON
parsing, no schema validation. This unblocks OPS-007 (multi-machine
credential sync) and any workflow involving non-JSON sensitive files.

## What's New

### Blob Encrypt/Decrypt (SC-010)

```bash
# Encrypt any file for one or more recipients:
seclusor secrets blob encrypt \
  --input ~/devsecops/vars/mattermost/agent-dispatch.sh \
  --output /tmp/agent-dispatch.sh.age \
  --recipient age1...

# Decrypt with an identity file:
seclusor secrets blob decrypt \
  --input /tmp/agent-dispatch.sh.age \
  --output ~/devsecops/vars/mattermost/agent-dispatch.sh \
  --identity-file ~/.config/seclusor/dispatch-identity.txt

# Works with passphrase-protected identities (SC-008):
seclusor secrets blob decrypt \
  --input /tmp/agent-dispatch.sh.age \
  --output ~/devsecops/vars/mattermost/agent-dispatch.sh \
  --identity-file ~/.config/seclusor/dispatch-identity.txt \
  --passphrase-env LANYTE_OPS_PASSPHRASE
```

**Design choices**:

- Input is opaque bytes — shell scripts, configs, binary tokens, TLS certs
- 10 MB default size limit with `--allow-large` override (guardrail)
- Decrypt output created with 0600 on Unix (decrypted content is sensitive)
- Atomic writes on decrypt via unique temp file + rename (no partial output)
- Silent success: exit 0, empty stdout (clean for scripting in loops)
- Multi-recipient encrypt, multi-identity decrypt

### Relationship to Existing Commands

| Command                  | Input format      | Validates schema | Use case                 |
| ------------------------ | ----------------- | ---------------- | ------------------------ |
| `secrets inline encrypt` | JSON secrets file | Yes              | Structured secret stores |
| `secrets bundle encrypt` | JSON secrets file | Yes              | Encrypted bundles        |
| `secrets blob encrypt`   | Any file          | No               | Opaque file encryption   |

## Other Changes

- Release workflow checksum generation now uses `find` instead of `ls`
  (shellcheck SC2012 fix)
- README updated with blob encryption and passphrase-protected identity
  feature bullets

## Security

- Blob decrypt uses `tempfile::NamedTempFile` with exclusive creation —
  no predictable temp paths, no symlink risk
- File size limit enforced at read time via bounded reader (no TOCTOU gap
  between metadata check and file read)
- Wrong identity produces a clear error without leaking plaintext

## Notes

- No breaking changes to the FFI contract or JSON schema
- Blob does not replace bundle or inline — those add JSON validation and
  credential-level features that blob deliberately omits
- Scrypt passphrase-mode data encryption (`--passphrase` on encrypt side)
  is tracked separately as SC-011
